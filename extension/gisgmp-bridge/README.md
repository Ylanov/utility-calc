# Мост ГИС ГМП → ЖКХ (`gisgmp-bridge`)

Браузерное расширение (Chrome/Yandex, Manifest V3). Раз в 12 часов под уже-
залогиненной ЭЦП-сессией пользователя читает раздел **«Начисления»** реестра
ГИС ГМП (`gisgmp.cgu.mchs.ru/charge/`) и отправляет непогашенные долги жильцов
в ЖКХ-биллинг (`asy-tk.ru`), во вкладку **«Долги 1С»**.

Только чтение реестра (GET). Никаких действий, паролей или ЭЦП наружу не передаётся.

## Как работает

1. Расширение постранично обходит `/charge/` под cookie-сессией пользователя.
2. Парсит строки: УИН, сумма, ФИО плательщика, назначение, статус квитирования/изменения.
3. Отбрасывает аннулированные, берёт только «Не сквитировано» (= долг).
4. Шлёт массив начислений в `POST /api/financier/gisgmp/sync` с токеном.
5. Бэкенд ЖКХ матчит ФИО → жильца и пишет долги 209 (комуслуги) / 205 (наём).

## Установка

1. На сервере ЖКХ задайте в `.env`:
   ```
   GISGMP_SYNC_TOKEN=<сгенерируйте: python -c "import secrets; print(secrets.token_urlsafe(32))">
   ```
   и передеплойте (push в main).
2. Скачайте папку `gisgmp-bridge` (или ZIP из вкладки «Долги 1С» → «Скачать расширение»).
3. В браузере: `chrome://extensions` → включить «Режим разработчика» →
   «Загрузить распакованное расширение» → выбрать папку `gisgmp-bridge`.
4. Откройте настройки расширения, впишите:
   - **URL ЖКХ** — `https://asy-tk.ru`
   - **Токен** — тот же `GISGMP_SYNC_TOKEN`
   - **URL реестра** — `https://gisgmp.cgu.mchs.ru`
5. Войдите в `gisgmp.cgu.mchs.ru` по ЭЦП и нажмите «Синхронизировать сейчас».

Дальше расширение синхронизирует само раз в 12 часов, пока браузер открыт и
сессия ГИС ГМП жива.

## Замечания

- Долг = сумма всех «Не сквитировано» начислений (кроме аннулированных), весь
  непогашенный остаток. Переплаты пока не считаются.
- Жильцов сопоставляем по ФИО (тот же матчер, что у импорта из Google Sheets:
  точное ФИО → инициалы → fuzzy → ручные алиасы). Не сопоставленные ФИО видны
  во вкладке «Долги 1С» → «Неразнесённые долги».
